Ростелеком же в свою очередь отказывается обслуживать/настраивать оборудование, которое <br> <b> "непредназначено" </b> для работы с IPTV</p>

## 🔹 Как установить?
Подключитесь по **SSH** к роутеру и выполните команду:


```
sh <(wget -O - https://raw.githubusercontent.com/v1rtuozz/openwrt-iptv-rostelecom/refs/heads/main/iptv-manager.sh)
```
<b> или </b>
```
wget -O /tmp/iptv-manager.sh https://raw.githubusercontent.com/v1rtuozz/openwrt-iptv-rostelecom/refs/heads/main/iptv-manager.sh && sh /tmp/iptv-manager.sh
```
Далее запустится менеджер по установке конфигурации
<br> <hr> <h3> Коды ошибок IPTV (Приставка WINK) </h3>
> [!CAUTION]
> <b> Ошибка 2-1 (No connection) </b> <br>
> Проверьте соединение кабеля от роутера до приставки. Вероятно проблема в соединении.

> [!CAUTION]
> <b> Ошибка 2-2 (DHCP) </b> <br>
> Неправильно настроенный Firewall или DHCP. <br>
> > Проверьте DHCP в веб-морде OpenWRT (192.168.1.1) <br>
> > <b> Network -> Interface -> iptv_lan -> DHCP Server </b> <br> 
> > Проверьте галочку в расширенных настройках на "Динамический IP". <br>

> [!CAUTION]
> <b> Ошибка 2-9-1 (Multicast unavalible) </b>
> <br> Неправильно настроенный IGMPProxy
> > Перейдите в панель (Status -> URL Log) <br>
> > При запуске канала на приставке, выловите IP. <br>
> > Пример: 2026-03-10 18:18:53,E4:27:71:26:90:16,192.168.100.144,52120,```212.12.12.236```,443,2,NONE,SSL,64,0,cnt-tula-itv03.svc.iptv.rt.ru
> > > Далее необходимо редактировать конфиг файл. <br>
> > > Заходим через SSH: ``` nano /etc/config/igmpproxy ``` <br>
> > > В конце параметра: <b> config phyint </b> указываем:
> > > ``` list altnet 'IP' ```
